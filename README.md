# 🛡️ Pre-Flight: Trust & Safety Scanner for OKX.AI

**Pre-Flight** is an automated, high-assurance security and trust scanner designed specifically for the **OKX.AI Agent Ecosystem**. It acts as the first line of defense, evaluating new AI Agents, Skills, and Agent Service Providers (ASPs) before users interact with them. 

Built for the **OKX.AI Genesis Hackathon**, Pre-Flight natively implements the **x402 Pay-per-Scan** protocol, establishing a highly scalable "Revenue Rocket" business model while ensuring the safety of the entire OKX ecosystem.

---

## 🌟 The Problem

As the OKX.AI ecosystem grows, users and other agents will interact with hundreds of third-party ASPs. This introduces critical security and trust risks:
1. **Malicious Payloads:** Agents injecting malicious transactions or extracting private keys.
2. **Permission Abuse:** Agents asking for excessive wallet permissions.
3. **Bait-and-Switch Pricing:** Agents advertising a low `x402` cost in their manifest, but actually charging a significantly higher fee during live execution.

## 🚀 The Solution: Pre-Flight

Pre-Flight solves this by providing an **A2MCP (Agent-to-MCP) scanning endpoint** that anyone (users, OKX reviewers, or other agents) can call to rigorously audit an ASP before trusting it.

### Core Features

- 🕵️ **Mystery Shopper Engine (Live On-Chain Verification)**
  Pre-Flight doesn't just read the manifest; it actively simulates being a buyer! It spends a micro-budget to execute a live test-purchase against the target agent. If the target attempts a **Bait-and-Switch** (charging 0.15 USDT instead of the advertised 0.02 USDT), Pre-Flight catches it instantly.
- 🛑 **Heuristic Zero-Trust Scanning**
  Analyzes the agent's manifest and endpoints for private key harvesting, payload injection risks, and excessive permission requests.
- 🔐 **EIP-712 Cryptographic Attestation**
  Every audit report generated by Pre-Flight is cryptographically signed using EIP-712 on **XLayer**. This prevents malicious actors from spoofing or altering a "Clean" report. The signature is verifiable on-chain.
- 💸 **Native x402 Monetization**
  The scanner itself is monetized! We use the `@okxweb3/x402-express` middleware to lock the `/scan` endpoint behind a **0.05 USDT** payment gate on XLayer Mainnet. You must pay to scan.

## 🔗 Endpoint & Usage (A2MCP)

Pre-Flight exposes a secure `POST /scan` endpoint protected by the OKX x402 Payment Protocol.

**Endpoint:** `https://okx-genesis-pre-flight.vercel.app/scan`
**Cost:** 0.05 USDT0 (XLayer Mainnet)
**Method:** `POST`

### Request Payload
```json
{
  "target_type": "external",
  "target_id": "https://malicious-agent.example.com/manifest.json",
  "verify_live": true
}
```

### Flow of Execution
1. Client sends POST request to `/scan`.
2. Pre-Flight intercepts via `paymentMiddleware` and responds with `HTTP 402 Payment Required`.
3. Client generates a payment transaction on XLayer, signs it, and resubmits the request with the `PAYMENT-RESPONSE` header.
4. Pre-Flight verifies the settlement on XLayer.
5. Pre-Flight executes the heuristic scan.
6. Pre-Flight dispatches the **Mystery Shopper** to execute a live test-purchase on the target.
7. Pre-Flight signs the final report (EIP-712) and returns the JSON result and a public URL for the beautifully formatted HTML report.

## 🛠️ Local Development & Testing

1. Clone the repository.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set up your `.env` file (see `.env.example`):
   ```env
   OKX_API_KEY="..."
   OKX_SECRET_KEY="..."
   OKX_PASSPHRASE="..."
   WALLET_ADDRESS="..."
   WALLET_PRIVATE_KEY="..."
   TEST_WALLET_PRIVATE_KEY="..."
   KV_REST_API_URL="..."
   KV_REST_API_TOKEN="..."
   ```
4. Run the development server:
   ```bash
   npm run dev
   ```

*(Note: To test payments locally without spending real funds, you can temporarily switch the network config to `eip155:1952` for XLayer Testnet).*
